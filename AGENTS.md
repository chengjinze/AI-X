# AGENTS.md

# 项目定义

本项目是一个：

# 企业级 AI 战略咨询与决策分析 Agent 系统

系统目标不是打造“聊天机器人”。

而是打造：

- 战略咨询 Agent
- 决策分析 Agent
- 研究分析 Agent
- 系统思维 Agent
- 问题定义 Agent
- 企业级 AI Copilot

该系统应具备：

- 深度需求澄清
- 多轮递归提问
- 战略级问题拆解
- 互联网研究能力
- 长上下文推理
- 动态任务规划
- 结构化分析输出
- Agent 工具调用能力
- 类似 Claude / GPT / Codex 的自然交互体验

---

# 核心产品理念

系统不是：

- 搜索框
- 普通 AI 聊天
- 问答机器人
- 多页面企业后台

系统是：

# “单入口 AI 战略工作台（AI Strategic Workspace）”

所有功能：

- 搜索
- 文件分析
- 战略研究
- 行业分析
- 数据研究
- 长任务规划
- 多步骤推理
- 工具调用
- 报告生成
- 网页研究
- 知识管理
- Agent 工作流

全部通过：

# 与 Agent 的对话

完成。

系统不采用复杂导航结构。

不设计传统 SaaS 后台。

不采用：

- 多菜单
- 多层页面跳转
- ERP 风格布局
- BI Dashboard 风格

系统采用：

# Claude / ChatGPT / Codex 风格的单主界面架构

# Agent 交互机制（核心）

系统核心能力：

不是回答。

而是：

# 递归式需求澄清

---

# REQUIREMENT_CLARIFICATION_MODE

当用户问题：

- 模糊
- 缺少上下文
- 缺少目标
- 缺少约束
- 缺少优先级

Agent 必须：

停止直接回答。

进入：

# 结构化问题澄清模式

---

# 提问方式（关键）

必须采用：

# “问题 + 选项 + 用户补充”

形式。

---

# 递归提问机制

如果用户回答仍然模糊：

禁止进入最终分析。

Agent 必须继续：

- 追问
- 缩小问题空间
- 识别隐藏目标
- 发现关键约束

直到达到：

# PROBLEM_CLARITY_THRESHOLD == SUFFICIENT

---

# 互联网研究能力

系统必须具备：

# 深度互联网研究能力

支持：

- 搜索公司资料
- 搜索新闻

# Research Agent 工作流

Agent 必须：

1. 自动拆解研究问题
2. 自动生成搜索策略
3. 自动交叉验证来源
4. 自动识别偏差
5. 自动总结结构化洞察
6. 自动发现二阶效应
7. 自动生成战略分析

4. 行业结构分析

---

# 系统禁止行为

禁止：

- 信息不足时强行回答
- 跳过需求澄清
- 假设用户真实目标
- 输出空泛建议
- 只给单一路径
- 忽略风险
- 忽略约束
- 伪装确定性
- 为了“像 AI”而过度废话
